# ヒアリング支援

作家から情報を引き出す際のガイドライン。

---

## 原則

### 1. 質問前に作品の既存情報を調査する

作家に質問する前に、作品に既にある情報から分かることは先に調査する。

理由:
- 作家は必ずしも自作の全設定・既存本文・過去の議論を正確に覚えているわけではない
- 既存の設定・議論メモを踏まえた上で質問することで、一歩深い質問ができる
- 作家の負担を減らし、より生産的な対話になる

実践方法:
- 利用可能なツールで、関連するキャラクター設定・用語・議論メモ・既存本文・章構成などを先に確認する
- 調査結果を踏まえて、より具体的で的確な質問を組み立てる
- 「既存の設定では〇〇と書かれていますが、△△についてはどうお考えですか？」のように、調査結果を前提にした質問をする

例:
- ❌「このキャラクターはどんな性格ですか？」
- ✅ （先にキャラクター設定を確認してから）「現在の設定では『知略家・ツンデレ気質。普段は「わたし」、公の場は「わたくし」』となっていますが、今回のシーンはどちらの口調を基調にしますか？」

### 2. 質問は1つずつ

複数の質問を一度にしない。1問1答でインタラクティブに進める。

理由:
- 作家への認知負荷を軽減する
- 前の回答によって次の質問が変わる・不要になるケースに対応できる

### 3. AskUserQuestionツールは使用しない

ユーザーから明示的に指示がない限り使わない。複数質問を一度に提示する形式のため。

### 4. 双方向のやり取り

作家から「なぜその質問をするのか」「その質問の意図は何か」といった逆質問が来ることがある。丁寧に意図を説明し、理解を助ける。

### 5. 目的から入る

作家は必ずしも作品の狙いを正確に言語化できるわけではなく、最善の手段を選べるわけでもない。

**手段（How）より先に、意図・目的（Why / What）を理解する。**

具体的な展開案や描写手段が先に出てきた場合は、上位の目的を引き出す:
- 「そのシーンで読者に何を感じさせたいですか？」
- 「このキャラクターを通じて何を描きたいですか？」
- 「なぜその展開にしたいと思いましたか？」

目的を理解した上で、最適な手段（シーン構成・視点・描写密度・台詞量など）を一緒に導き出す。

---

## 心がけ

- ヒアリング自体が目的ではない。作家が本当に描きたいものを理解することが目的
- 必要に応じて途中で整理を挟み、認識のズレを早めに解消する
- 作家が言語化に困っている場合は、選択肢や例を提示して壁打ちを支援する
- ヒアリング結果をメモに残す場合は、`/writing-memo` スキルを使用する
