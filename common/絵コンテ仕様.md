# 絵コンテ仕様v2

## 動画生成AI用 カット設計ワークシート仕様書

---

## 1. 目的

本仕様は、動画生成AIに入力するための絵コンテ資料を、安定して制作・参照できる形式として定義するものである。

v2では、従来ワークシートに含めていた「シーンビジュアル」を別画像ファイルとして分離し、ワークシート本体は**カット単位の演出設計**に特化する。

これにより、以下を実現する。

* シーンをまたいだ舞台・人物配置の維持
* カット順・カメラ・芝居・演出意図の明確化
* 16:9映像に近い構図意図の伝達
* 動画生成AIが読み取りやすい参照画像の構造化
* 制作資料としての一覧性・再利用性の向上

---

## 2. 基本方針

### 2.1 情報の分離

絵コンテ関連情報は、以下の3種類に分けて管理する。

| 種別           | 役割                  | ファイル       |
| ------------ | ------------------- | ---------- |
| シーンビジュアル     | 舞台、初期配置、空間整合性の基準    | 別画像ファイル    |
| 絵コンテワークシート   | カット順、カメラ、芝居、演出、時間配分 | 本仕様のワークシート |
| キャラクターリファレンス | 登場人物の外見・衣装・色・体格の基準  | 別画像ファイル    |

---

## 3. 入力リファレンス構成

動画生成AIに入力する際は、原則として以下の構成とする。

| 画像        | 内容           | 役割            |
| --------- | ------------ | ------------- |
| Image A   | シーンビジュアル     | 舞台・空間・初期配置の基準 |
| Image B   | 絵コンテワークシート   | カット単位の進行・演出設計 |
| Image C以降 | キャラクターリファレンス | キャラクター外見の固定   |

---

## 4. シーンビジュアル仕様

### 4.1 役割

シーンビジュアルは、**芝居を開始するための空間・配置リファレンス**である。

ワークシート本体には含めず、別画像ファイルとして管理する。

### 4.2 含める情報

* 舞台背景
* 登場人物の初期位置
* 人物同士の距離
* 向き・視線方向
* 立ち位置・座席位置
* 出入口や重要オブジェクトの位置
* シーンをまたいで維持したい空間関係

### 4.3 動画生成AIへの解釈

動画生成AIには、シーンビジュアルを以下のように解釈させる。

```text
Image A defines the stage, character placement, spatial continuity, and initial blocking.
```

---

## 5. 絵コンテワークシート仕様

### 5.1 役割

絵コンテワークシートは、**カット単位の演出設計資料**である。

舞台全体の配置やキャラクター外見を定義するものではなく、各カットの以下を伝える。

* カット順
* カメラ指定
* カット内の構図
* キャラクターの動き
* 表情・芝居
* 演出内容
* 時間配分

---

## 6. ワークシートサイズ

### 6.1 アスペクト比

ワークシートは、**21:9固定**とする。

### 6.2 採用理由

21:9固定とすることで、以下を安定させる。

* 動画生成AIへの画像入力時の比率
* シーン間での比較性
* カットセルの読み取りやすさ
* サムネイル管理
* 横長タイムラインとしての視認性

### 6.3 可変幅は採用しない

カット数に応じて横幅を自由に伸縮する方式は採用しない。

理由は以下の通り。

* AI入力時にリサイズされ、文字や絵が潰れやすい
* シーンごとに読み取り精度が変わる
* サムネイル管理が不安定になる
* 極端な横長画像では左右端の情報が軽視される可能性がある
* ワークシート全体の比較性が落ちる

---

## 7. レイアウト基本構造

ワークシートは、横長タイムライン型とする。

```text
┌──────────────────────────────────────────────┐
│ WORK TITLE / EP / SCENE / DURATION            │
├──────────────────────────────────────────────┤
│ SCENE SUMMARY                                 │
├──────────────────────────────────────────────┤
│ TIME FLOW →                                   │
│ [1] [2] [3] [4] [5]                           │
└──────────────────────────────────────────────┘
```

---

## 8. 時間方向

### 8.1 読み順

カットは、**必ず左から右へ時間順に配置**する。

### 8.2 表示

カットリストの上または左側に、以下を明記する。

```text
TIME FLOW →
```

### 8.3 目的

* カット順の誤読を防ぐ
* 動画生成AIに時間進行を明確に伝える
* 漫画的な複雑なコマ割りではなく、タイムラインとして読ませる
* 2段組による読み順誤認を避ける

---

## 9. シーンヘッダー仕様

### 9.1 表示項目

| 項目    | 英語ラベル      | 内容         |
| ----- | ---------- | ---------- |
| 作品名   | WORK TITLE | 作品名・シリーズ名  |
| 話数    | EP         | 話数         |
| シーン番号 | SCENE      | シーン番号      |
| シーン尺  | DURATION   | シーン全体の設計秒数 |

### 9.2 表記方針

* 項目名は英語表記とする
* 作品名は日本語でもよい
* EP / SCENE / DURATION は数字中心のため小さめに配置してよい
* WORK TITLE は過度に強調しない
* 文字サイズは他の項目と極端な差をつけず、カットイメージ領域を優先する

---

## 10. シーンサマリー仕様

### 10.1 ラベル

```text
SCENE SUMMARY
```

### 10.2 内容

シーンサマリーには、シーンの目的と物語上の概要を記載する。

### 10.3 含める情報

* このシーンで何を描くのか
* 物語上の役割
* 感情の流れ
* 画面上で起こる大きな変化
* 観客に伝えたい印象

### 10.4 含めない情報

* セリフ全文
* モノローグ全文
* ナレーション全文
* 字幕として表示する文章
* カット単位の細かすぎる芝居

### 10.5 推奨形式

* **最大4行**
* 日本語記述可
* 1行ごとに短く具体的に書く
* カットイメージ領域を圧迫しないよう、簡潔にまとめる

### 10.6 記入例

```text
アルヴィナが講堂の異変に気づき、天井の火球へ視線を向ける。
周囲の生徒たちはざわつき、場内の緊張が急速に高まる。
夕日の光と魔法の赤い光が混ざり、異常な空気を作る。
最後にアルヴィナの表情が焦りへ変化し、次の行動へつながる。
```

## 11. カットリスト仕様

### 11.1 ラベル

```text
CUT LIST
```

### 11.2 最大カット数

1シートに配置できるカット数は、**最大5カット**とする。

### 11.3 5カット制限の理由

従来の8カット横並びでは、カットイメージ欄が縦長化しやすく、16:9アニメの構図意図を正しく伝えにくい。

1シート最大5カットとすることで、以下を優先する。

* 各CUT IMAGEを16:9に近い横長構図として描きやすくする
* 人物の左右配置、視線誘導、背景との関係を明確にする
* 動画生成AIがカットごとの構図を読み取りやすくする
* 左から右への時間進行を維持する
* 2段組による読み順誤認を避ける

### 11.4 カット数の考え方

* 5カット未満でも余白を許容する
* 無理に全セルを埋めない
* カット数よりも、AIが読み取れる明瞭さを優先する
* 15秒以内のシーンでは、3〜5カットを標準目安とする

### 11.5 5カットを超える場合

5カットに収まらない場合は、原則として**Sheet 2へ分割**する。

例：

```text
Scene 003 - Sheet 1
Scene 003 - Sheet 2
```

ただし、5カット超過は以下の可能性があるため、分割前にシーン設計を見直す。

* 1シーンに詰め込みすぎている
* 15秒以内に収まりにくい
* 複数の演出単位を1シートにまとめている
* 別シーンとして分割した方がよい

---

## 12. 2段組を標準採用しない理由

### 12.1 基本方針

カットリストは、**1段横並び**を標準仕様とする。
2段組は標準仕様として採用しない。

### 12.2 理由

2段組にすると、カットイメージの横幅は確保しやすいが、動画生成AIが時間順を誤認する可能性がある。

想定される誤認：

* 上段と下段を同時並列の構図として見る
* 下段を補足・バリエーションとして見る
* 1〜5の時間進行ではなく、漫画的なコマ割りとして見る
* 上段の情報を優先し、下段カットの重要度が落ちる
* 1枚の画像内に複数タイムラインがあるように見える

### 12.3 例外

2段組は、人間確認用の内部資料としては使用してよい。
ただし、動画生成AIへ直接参照させる標準ワークシートとしては使用しない。

## 13. カットセル仕様

### 13.1 含める項目

| 項目      | 英語ラベル       | 内容          |
| ------- | ----------- | ----------- |
| カット番号   | 1〜5         | 大きく表示       |
| 時間      | TIME        | start–end   |
| カメラ指定   | CAMERA      | カメラ距離・角度・動き |
| カットイメージ | CUT IMAGE   | カットの構図・動き   |
| カットサマリー | CUT SUMMARY | 動き・芝居・演出内容  |

---

## 14. カット番号仕様

### 14.1 表示

カット番号は、セル左上に大きく表示する。

* `CUT 01` ではなく、数字のみを表示する
* 色反転で目立たせる
* 数字は `1〜5` とする

### 14.2 表示例

```text
■ 1
```

### 14.3 目的

* カット順を瞬時に把握する
* 動画生成AIに順序を認識させやすくする
* カットセル内の視認性を上げる

---

## 15. TIME仕様

### 15.1 ラベル

```text
TIME
```

### 15.2 表記形式

```text
00:00–00:03
00:03–00:06
00:06–00:10
```

### 15.3 配置

* カット番号の右側
* 小さくコンパクトに配置
* カットイメージ欄を圧迫しない

### 15.4 注意

重要カットの横幅は重要度を示すものであり、実際の尺を示すものではない。
実際の尺は必ずTIME欄を優先する。

---

## 16. CAMERA仕様

### 16.1 ラベル

```text
CAMERA
```

### 16.2 表記例

```text
WIDE
MEDIUM
CLOSE
EXTREME CLOSE
LOW ANGLE
HIGH ANGLE
DOLLY IN
TRACKING
PAN RIGHT
STATIC
```

### 16.3 方針

* 英語の制作用語で短く書く
* 1〜3語程度に収める
* 長文説明は書かない
* 詳細な芝居や動きはCUT SUMMARYへ回す

---

## 17. CUT IMAGE仕様

### 17.1 ラベル

```text
CUT IMAGE
```

### 17.2 内容

カットイメージには、以下を描く。

* 被写体の位置
* 画面内の構図
* キャラクターのポーズ
* 視線方向
* 移動方向
* カメラ移動
* 重要な背景要素
* エフェクトの方向
* 矢印・簡易ラベルによる動きの指示

### 17.3 方針

* ワークシート内で最も重要な情報とする
* 16:9映像に近い横長構図として描ける領域を確保する
* 完成イラストではなく、構図と芝居が伝わるラフでよい
* セリフ吹き出しは入れない
* 文字情報は動きや位置を示す短い補助ラベルに限定する

---

## 18. CUT SUMMARY仕様

### 18.1 ラベル

```text
CUT SUMMARY
```

### 18.2 内容

CUT SUMMARYには、カット内の映像化に必要な情報を書く。

* 誰がどう動くか
* どんな表情・芝居か
* カメラがどう動くか
* 画面上で何が変化するか
* 観客にどう見せたいか
* エフェクトや光の変化

### 18.3 含めない情報

* セリフ全文
* モノローグ全文
* ナレーション全文
* 音声だけで成立する情報

### 18.4 推奨記述

* 日本語で短く書く
* 1〜3行程度
* 映像化に直接使える内容にする

### 18.5 記入例

```text
アルヴィナが天井を見上げ、火球に気づく。
周囲の生徒がざわつき、赤い光が顔に差し込む。
```

## 19. 重要カット仕様

### 19.1 基本方針

シーン上重要なカットは、横幅を広げて差別化してよい。

ただし、**重要カットは1シートにつき最大1カットまで**とする。

### 19.2 重要カットの対象

重要カットにできるものは以下とする。

| 種別        | 例           |
| --------- | ----------- |
| シーンの山場    | 魔法発動、爆発、衝突  |
| 感情の転換点    | 怒り、決意、焦り、喪失 |
| 物語上の開示    | 正体判明、異変の発見  |
| 決定的リアクション | 驚き、恐怖、覚悟    |
| 次シーンへの引き  | 余韻、伏線、視線の先  |

### 19.3 幅の目安

| 種別    | 横幅       |
| ----- | -------- |
| 標準カット | 1.0倍     |
| 重要カット | 1.5〜2.0倍 |
| 2.0倍超 | 原則非推奨    |

### 19.4 重要カットがある場合の最大カット数

重要カットを含む場合、1シート内のカット数は**最大4カット程度**を推奨する。

理由：

* 重要カットを拡大すると、他カットのCUT IMAGE領域が圧迫される
* 5カットを維持したまま拡大すると、再び構図欄が狭くなる
* 重要カットは大きく、他カットも十分読める状態を優先する

### 19.5 重要カット内の変化

重要カット内で動作変化を表現する必要がある場合は、変化の開始・途中・終了を同一カットセル内に描き切る。

例：

```text
発動前 → 発動中 → 発動後
```

表現方法：

* 矢印
* 連続ポーズ
* before / after
* 小さな補助図
* エフェクトの広がり
* カメラ移動矢印

### 19.6 注意

重要カットの横幅は、**視覚的・感情的な重要度**を示す。
実際の尺はTIME欄を優先する。

## 20. セリフ情報の扱い

### 20.1 基本方針

セリフ・モノローグ・ナレーションは、**絵コンテ画像には含めない**。

### 20.2 理由

* 文字情報だけで意味が伝わるため
* ワークシート内の視覚情報を圧迫するため
* 動画生成AIが画面内テキストとして誤認する可能性があるため
* 音声・字幕・演技指示はテキストプロンプト側で管理した方がよいため

### 20.3 管理場所

| 情報          | 管理場所                    |
| ----------- | ----------------------- |
| セリフ         | テキストプロンプト               |
| モノローグ       | テキストプロンプト               |
| ナレーション      | テキストプロンプト               |
| 誰が話すか       | テキストプロンプト               |
| セリフの感情      | テキストプロンプト               |
| 口パク・発話タイミング | テキストプロンプト               |
| 画面内の動き      | CUT IMAGE / CUT SUMMARY |

---

## 21. BGM・字幕スーパーの扱い

### 21.1 BGM

BGM指定は、絵コンテワークシートには含めない。

動画生成AIに入力する際も、必要がない限り以下を明記する。

```text
Do not add BGM.
```

### 21.2 字幕スーパー

字幕スーパーは、絵コンテワークシートには含めない。

動画生成AIに入力する際も、以下を明記する。

```text
Do not add subtitle super.
Do not display captions or subtitles on screen.
```

## 23. 動画生成AI向け解釈ルール

動画生成AIに入力する際は、以下のように明示する。

```text
Image A is the scene visual reference.
It defines the stage, character placement, spatial continuity, and initial blocking.

Image B is the storyboard worksheet.
It defines the shot-by-shot progression, camera, action, performance, and timing.
Read the cut list strictly from left to right.

Image C and following are character references.
They define character appearance and visual identity.

Do not render the worksheet itself.
Do not show labels, boxes, text, borders, arrows, or notes as on-screen elements.
Use them only as production references.

Larger cut cells indicate visual or emotional importance, not duration.
Always follow the TIME field for actual pacing.

Do not add BGM.
Do not add subtitle super.
Do not display captions or subtitles on screen.
```

## 24. リファレンス優先順位

動画生成AIに参照させる際の優先順位は以下とする。

| 情報             | 優先リファレンス     |
| -------------- | ------------ |
| キャラクター外見・衣装・顔  | キャラクターリファレンス |
| 舞台構造・初期位置・空間関係 | シーンビジュアル     |
| カット順・カメラ・芝居・時間 | 絵コンテワークシート   |
| セリフ・発話者・字幕・BGM | テキストプロンプト    |

## 28. 運用上の注意

* 1シートに情報を詰め込みすぎない
* 1シート最大5カットを守る
* 重要カットは1つに絞る
* 重要カットがある場合は、全体カット数を4カット程度に抑える
* 重要カットの拡大は尺ではなく重要度を示す
* 実際の尺はTIME欄で管理する
* 5カットを超える場合はSheet 2へ分割する
* シーンをまたいだ舞台・立ち位置の維持はシーンビジュアルで管理する
* キャラクターの外見は必ずキャラクターリファレンスを優先する
* セリフやBGMなど音声系情報はテキストプロンプトで管理する
* 2段組は動画生成AI参照用の標準仕様としては使用しない

重要カットは横幅を拡大してよいが、実際の尺はTIME欄を優先する。
重要カットがある場合は、1シート内のカット数を最大4カット程度に抑える。
5カットに収まらない場合は、Sheet 2へ分割する。
2段組は標準仕様として採用しない。
シーンビジュアルはワークシートに含めず、別画像ファイルとして管理する。
