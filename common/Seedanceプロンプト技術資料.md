# Seedanceプロンプト技術資料

## 目的

この資料は、`labo/seedance/` に蓄積した参考プロンプトを分析し、Seedance向け動画生成プロンプトの書き方を共通技術として整理したものです。

対象は、物語やキャラクター固有の設定ではなく、複数作品で再利用できる以下の技術です。

- 生成結果の前提を固定する文脈設計
- カット、演技、演出を秒単位で伝える分解方法
- 表情、ポーズ、重心、衣服や髪の追従を高解像度に指定する方法
- コミカル演技、誇張物理、停止ポーズを安定して出すための書き方
- 今後の参考プロンプトを分析して蓄積するための観点

参考プロンプト本体は `labo/seedance/` に蓄積し、そこから抽出した共通原則をこの資料へ追記していく運用とします。

## 表現分類と使い分け

参考プロンプトから得た表現は、単語単位で切り貼りしない。必ず「何を見せたい場面か」「観客にどう感じさせたいか」「その表現が本来属するジャンルは何か」を先に決め、表現モードごとに束として使う。

特に、コミカルなシーンに対して、フォトリアル犯罪スリラー、実写的破壊、重いサバイバル、クールな殺陣の語彙を混ぜると、かわいさ、軽さ、安心感が壊れる。逆に、シリアスアクションに過剰なハート、星、ぐるぐる目を混ぜると、緊張感が壊れる。

### スタイルゲート

プロンプトを書く前に、必ず次のゲートを通す。

```text
PRIMARY MODE:
  このシーンの主表現モードを1つ選ぶ。

SECONDARY SUPPORT:
  補助表現を最大1つだけ選ぶ。主表現を壊さない範囲で使う。

DO NOT MIX:
  主表現と衝突する語彙を明示的に禁止する。

EMOTIONAL TARGET:
  観客に感じさせる主感情を1つ決める。
```

例:

```text
PRIMARY MODE: コミカルカートゥーン芝居
SECONDARY SUPPORT: 目的物ドタバタ
DO NOT MIX: フォトリアル時間停止スリラー、犯罪映画、実写的破壊、血や傷、クールな殺陣
EMOTIONAL TARGET: 怖い出来事を、かわいく安全なドタバタとして笑う
```

### コミカルシーンで使うべき表現

コミカルシーンでは、「危機そのもののリアルさ」ではなく、「危機を見たキャラクターの反応」と「安全なオチ」を主役にする。

使う表現:

- 目、眉、口、頬、肩、首の角度を使った顔芸。
- 0.1-0.25秒の完全停止ホールド。
- 予備動作、急加速、オーバーシュート、反動、急停止。
- 髪、袖、リボン、しっぽ、耳、装飾品の遅れ揺れ。
- 小さな汗、星、煙、ほこり、びっくり記号風の非文字エフェクト。
- 敵や相手も、怖さより「困惑」「空振り」「自滅」「食べ物に釣られる」などで処理する。
- 最後は安全、無傷、かわいい余韻で終える。

避ける表現:

- hyper-realistic practical destruction
- rigid body destruction
- brutal impact
- blood, gore, wounds
- grounded crime thriller
- raw desperate realism
- deafening explosionを主役にする演出
- 主人公が冷静すぎて顔芸しない演出
- カメラが激しすぎて表情と目的物が読めない演出

### クール表現を使ってよい場面

クール/スタイリッシュ表現は、キャラクターを「余裕がある」「強い」「怖いほど正確」「美しく危険」と見せたい場面に使う。

使ってよい例:

- 戦闘で敵の攻撃を無力化する。
- 魔法や剣技の必殺技を見せる。
- 悪役が威圧的に歩く。
- 主人公が恐怖ではなく余裕で危機を支配する。

コミカルシーンで一部だけ使う場合は、すぐに崩す。たとえば、ヒグマ登場を1秒だけローアングルで怖く見せても、その直後に鼻をひくひくさせる、蜂蜜に釣られる、転ぶ、困るなどでコメディへ戻す。

### フォトリアル時間停止表現の扱い

BP-015からBP-018の時間停止表現は、実写的で重い危機を描くための表現モードである。コミカルなアニメシーンにそのまま混ぜない。

使う場面:

- 実写風スリラー。
- 爆発、崩落、群衆、液体、破片などを精密に止める。
- 主人公だけが時間停止中に冷静に移動する。
- 音が爆音から絶対静寂へ切り替わる。

使わない場面:

- かわいいキャンプのドタバタ。
- 食べ物を奪い合う小さなギャグ。
- ぐるぐる目や汗を見せるカートゥーン。
- ゆるいキャラクター同士の安全な騒動。

もしコミカルに時間停止を使うなら、語彙を置き換える。

| 重い表現 | コミカル用の置き換え |
| --- | --- |
| hyper-realistic frozen debris | 小さな木の葉や食器が宙で止まる |
| absolute silence / bass drop | 一瞬だけ音がすぽんと抜ける |
| rigid body destruction | 小物がぷるぷる止まる |
| practical destruction VFX | かわいい漫画的な一時停止ギャグ |
| debris trajectories | ほこり、葉っぱ、食べ物の軌道 |

## BP-001の要点

### 1. 先に変えてはいけないものを固定している

冒頭で、参照画像のキャラクターをそのまま使うことを強く指定しています。

固定対象は以下です。

- 顔
- 髪型
- 服装
- 体型
- 色
- 比率
- キャラクターデザインの一貫性

動画生成では、動きや表情を細かく書くほどキャラクターの外見が崩れやすくなります。そのため、演技指示の前に「誰を、どの外見で、どこまで変えないか」を明記するのが有効です。

### 2. 世界、画角、カメラを狭く固定している

BP-001は、白背景、白い床、うっすら反射、柔らかい影、全身が常に見えるワイドショット、基本固定カメラという制約を置いています。

この制約の効果は次の通りです。

- 背景生成の揺れを減らす
- キャラクターの全身芝居を優先させる
- 足元、重心、鞄との位置関係を破綻しにくくする
- カメラ演出よりも芝居の読みやすさを優先する
- 短尺内で「何が起きたか」を視聴者に即時理解させる

複雑な演技を出したい場合は、背景やカメラを盛りすぎない方が安定します。

### 3. 動きの文法を明示している

BP-001の中核は、以下の運動文法です。

```text
静止
→ 予備動作
→ 一気に加速
→ 大きくオーバーシュート
→ ピタッと停止
→ 髪、袖、服、小物が遅れて揺れる
```

これはコミカル演技だけでなく、アクション、驚き、変身、魔法発動、転倒、リアクション全般に応用できます。

重要なのは「速く動く」とだけ書かないことです。速い動きを成立させるには、動く前のため、動いた後の行き過ぎ、停止、遅れて収まる副次運動まで書く必要があります。

### 4. 時間を感情と行動のビートで割っている

8秒を機械的な均等割りにせず、行動の意味ごとに区切っています。

- 0.0-0.8s: 疲れた導入
- 0.8-1.2s: 鞄の発見と驚き
- 1.2-1.6s: 左側へ高速移動して観察
- 1.6-2.0s: 右側へ高速移動して疑う
- 2.0-4.0s: 鞄を三段階で引っ張る
- 4.0-5.0s: 具合が悪そうな疲労
- 5.0-6.0s: 決意を立て直す
- 6.0-7.0s: 予想外に軽くなり回転
- 7.0-8.0s: 尻もちとオチ

尺の長い箇所は、単に長いのではなく「三段階の試行」「疲労の読ませ」「決意の間」など、観客に理解させる目的があります。

### 5. 表情を感情ラベルだけで終わらせていない

「驚く」「疲れる」「決意する」だけではなく、顔の構成要素に分解しています。

BP-001で使われている分解要素:

- 目の開き具合
- 瞳孔の大きさ
- 眉の角度と跳ね上がり
- 口の形
- 頬のふくらみ
- 顔色
- 目の焦点
- 口元の震え
- 頭の角度
- 首の引き
- 肩の高さ
- 呼吸による上半身の上下

表情は「感情名 + 顔の部位 + 変化の瞬間 + 停止ポーズ」で書くと読み取りやすくなります。

### 6. 停止ポーズを演出として扱っている

BP-001では、各アクションの終わりに0.1から0.2秒ほど止める指示が繰り返されます。

停止ポーズの役割:

- 感情を読み取る時間を作る
- シルエットを明確にする
- コミカルな間を作る
- スナップの効いた動きに見せる
- 生成モデルが次の動作へ流しすぎるのを防ぐ

短尺動画では、ずっと動かすよりも「完全に止まる瞬間」を入れた方が演技が伝わります。

### 7. 副次運動を細かく指定している

主動作だけでなく、髪、袖、スウェット、靴紐、アホ毛が遅れて揺れることを指定しています。

副次運動は、次の情報とセットにすると安定します。

- 何が揺れるか
- どの方向へ流れるか
- 主動作よりどれくらい遅れるか
- 最後にどう収まるか
- 誇張するが破綻させないこと

例:

```text
移動中は髪と袖が横方向に大きく流れる。
着地後、体はピタッと止まり、髪と袖だけが遅れて揺れ、最後にふわっと収まる。
```

### 8. 物理ルールとギャグルールを同時に指定している

BP-001では、鞄は最初まったく動かず、後半で突然軽くなります。一方で、足元や重心は破綻させないように制御しています。

このように、ギャグ表現でも以下を分けて書くと安定します。

- 現実より誇張してよい要素
- 絶対に破綻させない要素
- 動かしてよい物
- 動かしてはいけない物
- 途中で状態が変化する物

鞄のような小道具は、シーン内の物理ルールを明示しないと勝手に動いたり、消えたり、サイズが変わったりしやすいため、特に指定が必要です。

### 9. コミカル効果を控えめに制御している

漫画的な表現として、驚き記号、アホ毛のバネ、青い縦線、汗、星、ぐるぐる目、残像、モーションブラー、スピードラインが使われています。

ただし、プロンプトでは「控えめに」「一瞬だけ」「ホラーではなくコミカルで可愛い」と制御しています。

効果指定は、入れることだけでなく、強度と方向性も指定します。

```text
顔の横に小さな驚きエフェクトを一瞬だけ出す。
ホラーではなく、コミカルで可愛い誇張表現にする。
スピードラインは控えめに使う。
```

### 10. 最後に品質優先順位を再提示している

末尾の品質指定では、全体で守るべき優先順位を再度まとめています。

特に重要な優先順位:

- 表情演技を最優先する
- キャラクターデザインを安定させる
- 全身表示を保つ
- 動きは均一にしない
- 止まる部分は完全に止める
- 動く部分は一気に動かす
- 重要ポーズを必ずはっきり見せる

細かいタイムラインを書いた後に、最後に優先順位を再宣言すると、プロンプト全体の焦点が戻ります。

## 追加学習 2026-06-27

`labo/seedance/` に追加された7本の参考プロンプトから、BP-001だけでは不足していたカメラ、グラフィック、複数キャラ、音声同期の知見を抽出します。

### 1. カメラ方針は用途別に決める

BP-001では、全身芝居を安定させるために基本固定カメラが有効でした。一方、ダンス、PV、追いかけっこ、魔法少女登場演出では、カメラ自体を演出の主役として扱う方が強い結果になります。

用途別のカメラ方針:

| 用途 | 推奨カメラ | 狙い |
| --- | --- | --- |
| コミカル芝居 | 基本固定、ワイド、必要時だけ小さく寄る | 表情、全身、足元、小道具の破綻防止 |
| 可愛い一人ダンス | 2秒ごとのカメラ切替、正面/俯瞰/左右/背後 | 見飽きなさ、SNS短尺感、カメラ目線の親密さ |
| 高密度デュエットダンス | 常時マイクロズーム、ビートバウンス、ジッターパン | カメラも踊っている感覚、拍の視覚化 |
| 追いかけっこ | 低空追従、横トラッキング、前方回り込み | 速度感、臨場感、空間を駆け抜ける快感 |
| 複数キャラ登場 | 連続ロングテイク、回り込み、寄り引き | 各キャラのキメ紹介と最終配置への収束 |

重要なのは、ひとつのプロンプト内で「固定カメラ」と「一瞬も止まらないカメラ」を混在させないことです。どちらが主方針かを先に決めます。

### 2. 停止ポーズには二種類ある

BP-001の停止ポーズは、キャラクターもカメラも一瞬しっかり止める方式です。これは芝居やギャグに向いています。

追加資料では、ダンス/PV向けに「微振動ホールド」という別方式が見られます。決めポーズの構図は止まって見えるが、カメラだけは最小振幅で脈打ち続ける方式です。

使い分け:

- 完全停止ホールド: コミカルな間、驚き、疑い、疲労、オチ、感情読解に使う。
- 微振動ホールド: ダンスの決め、ライブ感、ビート感、カメラが生きている感覚に使う。

書き方:

```text
完全停止ホールド:
動作終わりでキャラクターとカメラを0.15秒完全に止め、表情を読ませる。

微振動ホールド:
決めポーズの構図は固定するが、カメラは最小振幅のマイクロズームとビートバウンスを続ける。
```

### 3. カメラを拍に同期させる

ダンス系プロンプトでは、カメラを単に「動かす」のではなく、身体パーツと拍に同期させています。

有効な同期例:

- 肩のヒットに合わせて、カメラが逆方向へ小さく弾く。
- 足のステップに合わせて、カメラが上下に小さくバウンスする。
- 指先やアームラインに合わせて、ズームが小刻みに追従する。
- ターン時は、カメラが逆回転で小さく回り込む。
- 強拍では、大きなスナップズームやホイップパンを重ねる。

この書き方は、ダンスだけでなく魔法発動、武器の振り、変身バンク、走り出しにも応用できます。

### 4. グラフィック要素はレイヤーとして管理する

BP-002とBP-005では、円、星、ハート、花、音符、矢印、リボン、ドット、チェック、波線などの2Dグラフィックが、キャラクターの動きに反応して出ます。

ここで重要なのは、装飾をただ増やすのではなく、レイヤー優先順位を明確にすることです。

推奨レイヤー:

```text
最優先: キャラクターの顔、目、手、重要ポーズ
次点: 身体の動線、髪、衣装、シルエット
補助: 2Dグラフィック、図形、キラキラ、速度線
背景: 白背景、教室、スタジオ、抽象模様
```

グラフィック指定では、以下を必ず書きます。

- 形: 丸、星、ハート、花、音符、矢印、波線など。
- 質感: フラット塗り、クレヨン、オイルパステル、シャープエッジなど。
- 色: パステル、ピンク、水色、黄色、白、ラベンダー、ミントなど。
- 動き: 出現、バウンス、拡大縮小、回転、時間差、ふわっと消える。
- 遮蔽制御: 顔や重要部位を隠しすぎない。

### 5. カメラ目線はカメラごとに指定する

一人ダンス系では、単に「カメラ目線」と書くだけでは足りません。正面以外のカメラでも、顔の向き、首のひねり、視線の返しでレンズを見ていることを明記します。

例:

```text
背後カメラでは、肩越しに振り返ってカメラ目線にする。
左カメラでは、身体は左向きのまま、顔と視線だけをレンズへ返す。
俯瞰カメラでは、見上げるようにカメラ目線を保つ。
```

カメラ切替を使う場合は、各カメラに「その画角だから成立する仕草」を割り当てます。背後カメラなら振り返り、俯瞰なら見上げ、正面ならウィンクや指差しが向いています。

### 6. 追いかけっこは地形をアクション設計に入れる

BP-006では、住宅街そのものがアクションの設計要素になっています。細い路地、ブロック塀、階段、駐輪場、段差、曲がり角を、猫と少女の動きに結びつけています。

追いかけっこや移動アクションでは、背景をただの場所として書かず、障害物と動線として書きます。

```text
猫は駐輪場脇をすり抜ける。
少女は急な方向転換で追う。
猫は塀の上に飛び乗り、少女は階段と近道を使って食らいつく。
曲がり角を抜けると視界が開け、最後に速度を落として余韻を作る。
```

この型は、森、学園、洞窟、市街地、室内逃走などにも使えます。

### 7. 複数キャラはスロットと役割で固定する

BP-007では、複数の参照画像を単に並べるのではなく、各画像にスロットと役割を割り当てています。

有効な役割:

- `[FIRST FRAME][WORLD/BG ANCHOR]`: 開始時の世界、背景、光、床、空気感。
- `[CHARACTER: 名前]`: 各キャラクターの外見、配色、位置、モチーフ。
- `[LAST FRAME][POSE/LAYOUT ANCHOR]`: 最終集合ポーズ、立ち位置、向き、構図。
- `[VOICE REFERENCE]`: 台詞の声質。

複数キャラでは、以下の失敗を明示的に禁止します。

- 混ざる
- 入れ替わる
- 平均顔になる
- モーフする
- 途中で消える
- 再入場する
- 位置がズレる
- 配色やモチーフが伝播する

### 8. ロングテイクでは「出現後の継続」を書く

複数キャラの登場演出では、キャラクターが一人ずつ出るだけでは不十分です。一度現れたキャラが最後までその場に残り、位置を保持することを明記します。

```text
各少女は一人ずつ加算される。
一度現れたら最後までその場に残る。
誰も消えない、再入場しない、位置がズレない。
最終的にLAST FRAMEの配置と向きへ収束する。
```

この考え方は、チーム集合、パーティ紹介、敵幹部紹介、ダンス隊形、家族集合カットでも有効です。

### 9. 音と台詞のポリシーを先に決める

参考プロンプトには、`BGMなし。SEのみ。` のものと、SeedanceにBGM自動生成を任せるものがあります。どちらも有効ですが、曖昧にしないことが重要です。

音声ポリシーの選択肢:

- BGMなし、SEのみ。
- BGMは生成しない、口パクも不要。
- BGMはSeedanceに自動生成させる。
- 台詞は最後だけ入れる。
- 台詞は演技キューのみで画面に文字表示しない。

台詞がある場合は、順序とポーズ同期を明記します。

```text
「Yes!」が先、その後に「フォロワーゴーゴーゴー！」。
「Yes!」はポーズ前の溜め。
決めポーズと同時に二つ目の台詞を言い切る。
字幕、吹き出し、画面内テキストは出さない。
```

### 10. 回避事項は「起きやすい失敗」を具体的に書く

追加資料では、回避事項が非常に具体的です。単に「品質を保つ」ではなく、モデルが起こしやすい破綻を列挙しています。

具体例:

- 台詞順序が入れ替わる。
- 台詞が早口で潰れる。
- 決めポーズと台詞のタイミングがズレる。
- 実際に変身して衣装が変わる。
- 5人が混ざる、入れ替わる、平均顔になる。
- 2D指定なのに3D化、実写化する。
- カメラの手ブレが激しすぎて酔う。
- エフェクトが顔を隠す。

回避事項は、品質指定の裏返しではなく、生成モデルの典型的な失敗リストとして書くと効果的です。

## 追加学習 2026-06-28

`labo/seedance/` に追加された7本の参考プロンプトから、アクション、ストーリーボード参照、VFXの因果連続性、目的物固定、Vlog風の非映画的表現に関する知見を抽出します。

### 1. ストーリーボード参照は「描く対象」ではなく「撮影指示」として読む

BP-009では、ストーリーボード参照を画面に再現するのではなく、ショット順、カメラ文法、ステージング、画面方向、アクションの連続性を読むためのアンカーとして扱っています。

有効な書き方:

```text
Use the storyboard reference as the visual storyboard anchor for shot order, camera grammar, staging, screen direction, and payoff continuity.
Render the final filmed scene, not the storyboard sheet or sketch style.
Read each panel as a separate shot sample of the same scene.
Do not merge panel poses.
```

さらに、キャラクター参照とストーリーボードの優先順位を分けます。

- キャラクター参照: 顔、体型、衣装、素材、最終外見の唯一の権威。
- ストーリーボード: 構図、ポーズ、動線、カメラ、ショット順の参照。
- ストーリーボード内のラフなシルエット: 外見固定には使わない。

これは、絵コンテワークシートやラフスケッチをSeedanceに渡すときの基本原則として有効です。

### 2. 変換ギミックは「入力、変換、出力、継続」を固定する

BP-009の強い点は、弾丸が単に消えるのではなく、キャラクターの輪郭に当たった瞬間に花へ変わり、花びらが空間に残り、最後のオチまでつながることです。

変換ギミックを書くときは、次を固定します。

```text
入力: 何が飛んでくるか。
接触点: どこに当たるか。
変換: 何へ変わるか。
出力: 変換後の形、色、粒子、破片、音。
継続: 変換後のものが次のショットでどう残るか。
回収: 最後のオチや決めにどうつながるか。
```

例:

```text
Bullets strike the coat outline and pop into tiny red, yellow, blue, and white blossoms.
Petals kick outward and remain in the lane as later shots continue through flower-and-casing rain.
```

魔法、変身、攻撃無効化、食べ物ギャグ、被弾表現を安全に置き換える演出に応用できます。

### 3. アクションはフェーズ単位で「溜め、解放、残響」に分ける

BP-010は6秒の短尺ながら、抜刀を次の三段階に分けています。

- Phase A: 抜刀の溜め。
- Phase B: 爆発的突進。
- Phase C: 空を裂く斬撃線と残痕。

短い必殺技や魔法発動では、秒単位のカットだけでなく、フェーズ名をつけると意図が安定します。

```text
Phase A: エネルギーが集まる。身体が逆方向へねじれ、重心が沈む。
Phase B: 身体が解き放たれ、カメラへ突進し、主動作がフレームを通過する。
Phase C: 結果だけが画面に残り、光線、残痕、煙、花びらなどが収束する。
```

重要なのは、技の瞬間だけでなく、直前の身体のねじれ、重心、エネルギー臨界、通過後の残響まで書くことです。

### 4. 目的物は「画面上の欲望の中心」として固定する

BP-011とBP-012では、金のリンゴが全カットの目的物です。柱、台座、床、リンゴの位置関係が全カットで一貫し、リンゴは常に重要に見えるよう指定されています。

目的物があるプロンプトでは、以下を明記します。

- 目的物の外見、色、発光、サイズ。
- どこに置かれているか。
- 背景や人物との位置関係。
- いつ重要に見せるか。
- 消えない、複製しない、別物にならないこと。
- 目的物を見た瞬間の表情ラッシュ。

例:

```text
The golden apple is the visual objective of the scene.
Keep the columns, pedestal, floor, and apple positions consistent in every shot.
The apple must remain visually important whenever the character's attention shifts toward it.
```

この考え方は、キャンプの焼き魚、蜂蜜瓶、魔法アイテム、武器、手紙、宝石などにそのまま使えます。

### 5. 表情ラッシュは「全編」ではなく「見せ場」に集中させる

BP-011では、重要な見せ場で4から6種類の表情を短く切り替えています。一方で、全編ずっと細かく変化させ続けるのではなく、見せ場を明確に分けています。

有効な考え方:

- すべての瞬間を細かくしすぎない。
- 重要な瞬間だけ表情と動きを爆発させる。
- 見せ場ごとに3から6種類の表情を用意する。
- 中間を滑らかにつなげすぎず、パッ、パッ、パッと切り替える。
- 同じ姿勢のまま表情だけ変えない。首、肩、腰、手、指、膝、つま先も変える。

表情ラッシュの例:

```text
疑い顔 -> 驚愕顔 -> 星目の感動顔 -> ぽかん顔 -> うっとり顔 -> 悪だくみのにやり顔
```

### 6. 二次動作は部位ごとに遅延時間と減衰を指定する

BP-012では、長い髪、袖、スカート、腰の装飾布、タッセル、装飾品などの遅れ揺れが細かく指定されています。

二次動作を書くときは、単に「揺れる」と書かず、以下を指定します。

- どの部位が遅れるか。
- 何秒遅れるか。
- どの方向に弾むか。
- 何回反動するか。
- どのように収まるか。
- 衣装、体型、装飾品を破綻させないこと。

上品さが必要な身体・衣装の二次動作では、露出、衣装ズレ、脱衣、肌の強調を明示的に禁止します。二次動作は、髪や袖と同じくアニメ的なフォロー・スルーとして扱います。

### 7. 敵や相手役にも感情曲線を与える

BP-009では、C1が余裕のある捕食者のように進む一方で、C2は自信からパニックへ崩れていきます。アクションでは主人公だけでなく、相手役の心理変化を書くと画面が読みやすくなります。

例:

```text
C1 moves like an amused predator under impossible pressure.
C2's confidence fractures into panic.
```

敵、怪物、ヒグマ、群衆などにも「何を見て、どう表情が崩れ、どう行動が変わるか」を与えると、ギャグやアクションの因果が強くなります。

### 8. 安全なアクションでは「無力化の方法」を指定する

BP-008とBP-009では、攻撃的な状況でも、銃を蹴り上げる、武器を払い落とす、敵同士が自滅する、花へ変換するなど、暴力表現をコントロールしています。

安全寄りのアクションでは、次のように書くとよいです。

- 武器を払い落とす。
- 相手が自分の勢いで転ぶ。
- 攻撃が花、煙、黒い霧、紙吹雪などへ変わる。
- 痛そうな描写ではなく、コミカルなリアクションで済ませる。
- 血、傷、切断、露骨な苦痛を禁止する。

コミカルなヒグマ、魔物、ギャング、怪物にも応用できます。

### 9. Vlog風では「綺麗に撮らない指示」が品質になる

BP-014は、映画的な完成度ではなく、2000年代後半のデジタルカムコーダーVlogらしさを狙っています。この場合、普通のプロンプトで避けがちな不完全さが品質指定になります。

Vlog風の有効要素:

- 目に見える手持ち揺れ。
- 不完全なフレーミング。
- 時折のフォーカスハンティング。
- エクスポージャーブリージング。
- わずかなデジタルグレイン。
- 暖かくフェードした色調。
- 決して中央に置きすぎない。
- 決してポーズを取りすぎない。
- 自然な環境音のみ。
- 音楽なし、映画的スタイリングなし、広告的ルックなし。

ただし、Vlog風でも参照キャラの顔、体型、服装、肌色、髪型は固定します。「不完全に撮る」と「参照が崩れる」は別です。

### 10. マクロアクションは三段階でスケールを上げる

BP-013のようなブロックバスター系では、細かい表情よりもスケールの段階上げが重要です。

```text
1. 軌道降下: 大気圏、火、衝撃波、カメラ追従。
2. 戦場突破: 着地、群れの突破、巨大爪の回避、爆発。
3. 最大火力: オーバーロード、主砲、巨大敵撃破、孤立した勝利シルエット。
```

この型は、短尺トレーラーや大型バトルの概要プロンプトに向きます。ただし、キャラクター演技を細かく制御したい場合は、BP-001やBP-011のような秒単位の演技設計を優先します。

## Seedance向けプロンプトの基本構造

BP-001からBP-018までの参考資料を統合した推奨構造は以下です。

```text
0. 表現モードと混入禁止
1. 参照固定
2. 参照画像の役割スロット
3. ストーリーボードやアンカー画像の解釈
4. 画面世界とスタイル
5. カメラ方針
6. モーション文法
7. 目的物と因果連続性
8. グラフィック/エフェクトレイヤー
9. 物理ルールと小道具ルール
10. 音と台詞のポリシー
11. 尺と全体方針
12. 秒単位のタイムライン、またはフェーズ設計
13. 品質指定と優先順位
14. 回避事項
```

### 0. 表現モードと混入禁止

最初に、主表現モードを1つ決めます。補助表現は最大1つだけにし、主表現を壊す語彙を明示的に禁止します。

```text
PRIMARY MODE: コミカルカートゥーン芝居
SECONDARY SUPPORT: 目的物ドタバタ
DO NOT MIX: フォトリアル時間停止スリラー、クール殺陣、実写的破壊、犯罪映画の重さ
EMOTIONAL TARGET: 安全でかわいいドタバタを笑う
```

### 1. 参照固定

キャラクター参照や背景参照がある場合、最初に固定します。

```text
Image1のキャラクターをそのまま使用する。
顔、髪型、髪色、目の色、服装、体型、比率、アクセサリを参照画像から変えない。
動きや表情は大きく変えてよいが、キャラクターデザインは一貫させる。
```

### 2. 参照画像の役割スロット

参照画像が複数ある場合は、各画像が何を固定するかを先に割り当てます。

```text
Image A is [WORLD/BG ANCHOR]: 背景、光、床、空気感、開始構図を固定する。
Image B is [CHARACTER: 名前]: 顔、髪、衣装、体型、色、モチーフを固定する。
Image C is [LAST FRAME / POSE ANCHOR]: 最終ポーズ、立ち位置、向き、構図を固定する。
Image D is [VOICE REFERENCE]: 台詞の声質だけを参照する。
```

複数キャラの場合は、キャラクターごとに名前、位置、色、モチーフを分けて書き、混ざり、入れ替わり、平均化、モーフを禁止します。

### 3. ストーリーボードやアンカー画像の解釈

絵コンテやラフ画像を渡す場合は、画像自体を描かせるのではなく、撮影指示として読むことを明記します。

```text
Storyboard reference is the anchor for shot order, camera grammar, staging, screen direction, and action continuity.
Render the final filmed scene, not the storyboard sheet, sketch style, labels, borders, or text.
Character reference images are the sole authority for final appearance.
Storyboard silhouettes are staging references only.
```

### 4. 画面世界とスタイル

生成する映像の見た目を先に固定します。

```text
高品質な3DCGカートゥーンアニメーション。
セルルック寄りの3D。
柔らかいスタジオライティング。
背景は白い3D空間。
床には柔らかい影とうっすらした反射がある。
```

Vlog風やホームビデオ風では、あえて映画的にしない指示もスタイルです。

```text
Visible handheld shake, imperfect framing, occasional focus hunting, exposure breathing, slight digital grain.
Do not center the subject perfectly. Do not make it look like a fashion commercial or cinematic ad.
```

### 5. カメラ方針

演技を優先する場合は、カメラを制限します。

```text
キャラクターの全身が常に見えるワイドショット。
カメラは基本固定。
必要な場面だけ、わずかなズームや手持ち感で勢いを補助する。
足元、重心、小道具との距離関係が常に分かるようにする。
```

ダンス、PV、追いかけっこでは、カメラを拍や身体の動きに同期させます。

```text
カメラはビートに合わせて軽く上下にバウンスする。
肩のヒットに合わせて小さく逆方向へ弾く。
指先の軌道へ短いズームで追従する。
決めポーズでは構図を保ったまま、最小振幅の微振動ホールドを続ける。
```

### 6. モーション文法

動きの質を一括指定します。

```text
アニメーションはきびきびした緩急を重視する。
各動作は「静止→予備動作→一気に加速→オーバーシュート→ピタッと停止→髪と服が遅れて揺れる」の順で見せる。
動きは均一にしない。
止まる部分は完全に止め、動く部分は一気に動かす。
```

### 7. 目的物と因果連続性

小道具、宝物、食べ物、武器、魔法効果など、画面上の目的物がある場合は一貫性を固定します。

```text
The golden apple / honey jar / fish / weapon is the visual objective of the scene.
Keep its position, size, color, and relationship to the characters consistent.
It must not disappear, duplicate, transform into an unrelated object, or lose importance.
```

変換ギミックがある場合は、入力、接触点、変換、出力、継続、回収を書きます。

```text
Bullets strike the coat outline and transform into flower blossoms.
The petals remain in the air and continue into later shots as flower-and-casing rain.
```

### 8. グラフィック/エフェクトレイヤー

2Dエフェクトや図形を使う場合は、形、質感、色、動き、遮蔽制御をまとめます。

```text
AE風の2Dフラットアニメーション。
クレヨンやオイルパステルのような手描き感。
丸、星、ハート、花、音符、リボン、波線、ドットを使う。
配色はピンク、水色、黄色、白、ラベンダー、ミントグリーン。
エフェクトは手、顔まわり、足元の動きと同期して出現し、バウンスしてふわっと消える。
顔、目、手、重要ポーズを隠しすぎない。
```

### 9. 物理ルールと小道具ルール

破綻しやすい対象を先に制御します。

```text
床との接地、足元、重心は破綻させない。
現実より少し誇張したカートゥーン物理にする。
小道具は指定があるまで動かさない。
小道具のサイズ、色、位置関係を途中で変えない。
```

二次動作は、部位、遅延、反動、収束まで書くと安定します。

```text
髪、袖、スカート裾、装飾布、タッセル、アクセサリは本体より0.05-0.15秒遅れて揺れる。
急停止後に2-3回小さく反動し、短い減衰揺れで収まる。
衣装のズレ、脱衣、過度な露出、体型や装飾の破綻は禁止する。
```

### 10. 音と台詞のポリシー

音の扱いを先に固定します。

```text
BGMなし。SEのみ。
または、BGMはSeedanceが自動生成する。
台詞は最後だけ入れる。
台詞は画面内テキストとして表示しない。
台詞の順序、言い切るタイミング、ポーズとの同期を固定する。
```

### 11. 秒単位のタイムライン、またはフェーズ設計

各時間帯は、以下の順で書くと安定します。

```text
[開始-終了秒]:
この区間の目的。
開始姿勢。
視線。
予備動作。
主動作。
表情変化。
停止ポーズ。
副次運動。
必要なエフェクト。
破綻防止の注意。
```

必殺技や短いアクションでは、フェーズ設計も有効です。

```text
Phase A: 溜め。身体のねじれ、重心、エネルギー蓄積。
Phase B: 解放。爆発的加速、カメラ通過、主動作。
Phase C: 残響。光線、残痕、花びら、煙、余韻の収束。
```

### 12. 品質指定と優先順位

末尾で、モデルに優先してほしいことをまとめます。

```text
品質指定:
表情演技を最優先する。
キャラクターデザインを安定させる。
ポーズはシルエットが分かりやすいものにする。
重要な表情とポーズは0.1から0.2秒しっかり止める。
全身表示を維持する。
小道具の位置と状態を一貫させる。
```

### 13. 回避事項

生成モデルが起こしやすい破綻を具体的に書きます。

```text
キャラクターを混ぜない。
衣装を変えない。
参照画像の色やモチーフを他キャラへ伝播させない。
エフェクトで顔を隠さない。
カメラを酔うほど揺らさない。
台詞順序を入れ替えない。
字幕、吹き出し、ロゴ、ウォーターマークを出さない。
```

## カット記述テンプレート

Seedance向けの各カット、各時間帯は次のテンプレートで書けます。

```text
[0.0-0.0s]:
目的:
  この時間帯で観客に理解させたいことを書く。

開始状態:
  キャラクターの位置、姿勢、視線、小道具との距離を書く。

予備動作:
  主動作の前に、逆方向への沈み込み、目線移動、息を吸う、肩を上げるなどを書く。

主動作:
  何が、どの方向へ、どの速度で、どれくらい動くかを書く。

表情:
  感情名だけでなく、目、眉、口、頬、顔色、頭の角度、肩の高さを書く。

停止ポーズ:
  動作終わりのシルエットと、何秒止めるかを書く。

副次運動:
  髪、袖、服、リボン、小物などが遅れてどう揺れ、どう収まるかを書く。

エフェクト:
  必要な場合のみ、残像、モーションブラー、汗、星、記号などを書く。強度も指定する。

物理と継続:
  足元、重心、小道具、背景、キャラクター外見の破綻防止を書く。
```

## 表情指定の分解表

| 感情 | 目 | 眉 | 口 | 体 | 追加要素 |
| --- | --- | --- | --- | --- | --- |
| 驚き | 大きく見開く、瞳孔が小さくなる | 勢いよく跳ね上がる | 小さな「え？」から丸く開く | 上半身が一瞬後ろにのけぞる | 小さな驚き記号、髪や袖が遅れて持ち上がる |
| 疑い | 目を細める | 片眉を上げる | 小さくへの字 | 前かがみで観察する | 顎に手、足先で床を小さく叩く |
| 疲労 | 半目、焦点が合わない | 下がる | 開いて荒い息 | 膝に手、肩が上下、背中が丸い | 汗、青白い影、青い縦線、乱れた前髪 |
| 決意 | 目に力が入る | キリッと下げる | 引き結ぶ | 足を肩幅、低い姿勢 | 袖をまくる、カメラがわずかに寄る |
| パニック | ぐるぐる目、目線が定まらない | 上がる、歪む | 開く | 体がのけぞる、回転する | 円形ブラー、残像、遠心力で髪と袖が広がる |
| 混乱 | ぼーっとした目 | 緩む | 小さく開く | 首を傾ける、尻もち | 星が回る、ラストで長めに停止 |

## モーション指定の分解表

| 動き | 書くべき要素 | 例 |
| --- | --- | --- |
| 歩く | 速度、疲労感、肩、視線、髪と服の揺れ | 肩を落とし、目は半開きで、歩くたびに袖が遅れて揺れる |
| 急停止 | どの姿勢で止まるか、止まる長さ | 片足を少し浮かせたままピタッと停止し、0.15秒固まる |
| 高速移動 | 予備動作、移動方向、残像、着地、行き過ぎ | 逆方向へ沈み込み、横へスナップ移動し、少し行き過ぎて戻る |
| 引っ張る | 回数、強度差、体の伸び、足の滑り、小道具の状態 | 1回目は軽く、2回目は腰を落とし、3回目は全力。鞄は完全固定 |
| 倒れそうになる | 揺れ幅、踏ん張り、重心 | 左右にふらつくが、膝に手をついたまま踏ん張る |
| 回転する | 回数、接地、髪と服の遠心力、表情変化 | 足は床についたまま4回転し、髪が円を描き、目がぐるぐるに変わる |
| 着地する | 衝撃、潰れ、追従、余韻 | 尻もちの瞬間に体が少しつぶれ、髪が一拍遅れて落ちる |

## 良いSeedanceプロンプトの条件

- 参照画像から変えない要素を冒頭で固定している。
- 複数参照の場合、各画像の役割がスロットとして明確である。
- 絵コンテやラフ画像を使う場合、撮影指示として読むことを明記している。
- 背景、画角、カメラ、尺が先に決まっている。
- カメラ方針が、固定、切替、追従、微振動、ロングテイクのどれかに整理されている。
- 「何が起きるか」だけでなく「どう動くか」が書かれている。
- 動作に、予備動作、加速、オーバーシュート、停止、追従がある。
- 必殺技や短いアクションでは、溜め、解放、残響のフェーズがある。
- 表情が、目、眉、口、頬、顔色、頭、肩まで分解されている。
- 表情ラッシュを全編ではなく重要な見せ場に集中させている。
- 感情変化の瞬間に0.1から0.2秒の停止ポーズがある。
- ダンス/PVでは、微振動ホールドやカメラの拍同期が設計されている。
- 2Dエフェクトは形、質感、色、動き、遮蔽制御が明確である。
- 小道具の状態と物理ルールが明確である。
- 目的物がある場合、位置、サイズ、重要度、消失/複製禁止が明確である。
- 変換ギミックがある場合、入力、接触点、変換、出力、継続、回収が明確である。
- エフェクトは強度、時間、意味が指定されている。
- 二次動作は部位、遅延、反動、収束、破綻禁止が明確である。
- 音、SE、BGM、台詞、字幕禁止の扱いが明確である。
- 複数キャラでは、混同、入れ替わり、平均化、モーフ、位置ズレを禁止している。
- Vlog風など特殊な質感では、あえて不完全に撮るための制御がある。
- 最後に品質指定と優先順位をまとめている。
- 最後に起きやすい失敗を具体的な回避事項としてまとめている。

## 避けるべき書き方

| 避ける表現 | 問題 | 改善例 |
| --- | --- | --- |
| かわいく驚く | 表情と動作が曖昧 | 目を大きく見開き、眉が跳ね上がり、口が丸く開く。上半身が一瞬のけぞる |
| 素早く動く | 速度だけで芝居が出ない | 逆方向へ沈み込み、次の瞬間に横へスナップ移動し、着地後少し行き過ぎて戻る |
| 疲れる | 画として何を出すか不明 | 膝に手をつき、肩が大きく上下し、半目で焦点が合わず、額に汗が出る |
| コミカルにする | モデル任せになりやすい | ぐるぐる目、小さな星、控えめな残像、最後に0.5秒のオチの間を入れる |
| カメラを派手に動かす | 演技や足元が崩れやすい | 基本固定、決意の場面だけごくわずかに寄る |
| ずっと滑らかに動く | メリハリが消える | 止まる部分は完全に止め、動く部分は一気に動かす |
| カメラをいい感じに動かす | どの方向の運動か不明 | 2秒ごとに正面、俯瞰、左、右、背後へ切り替える。各カメラで目線を返す |
| 可愛いエフェクトを出す | 形、質感、遮蔽が曖昧 | クレヨン風のハート、星、花を手元に出し、バウンスして消える。顔は隠さない |
| 5人を登場させる | キャラ混同や位置ズレが起きやすい | 各ImageにCHARACTERスロットを割り当て、最終配置をLAST FRAMEアンカーで固定する |
| 台詞も入れる | 順序や同期が崩れやすい | 台詞順、発話タイミング、決めポーズとの同期、字幕禁止を明記する |
| storyboardを参考にする | シート自体やラフ絵柄が出る | ショット順、カメラ、ステージングだけを読み、最終映像として描く |
| 魔法で変える | 何がどう変わるか曖昧 | 弾丸が輪郭に当たり、花へ変わり、花びらが次ショットまで残る |
| すごい必殺技 | 尺内の構造が曖昧 | Phase A 溜め、Phase B 突進、Phase C 斬撃残痕の収束に分ける |
| Vlog風にする | ただ低品質になる | 手持ち揺れ、フォーカスハンティング、露出ブリージング、不完全フレーミングを意図的に指定する |

## 参考プロンプトを追加したときの分析手順

新しい良い参考プロンプトを見つけたら、次の手順で学習します。

1. 参考プロンプト本体を `labo/seedance/` に保存する。
2. ファイル名は `bestpractice_2.md` のように連番、または内容が分かる名前にする。
3. この資料の「参考資料」表に追記する。
4. 以下の観点で分析する。

```text
表現モード:
  どのモードに属するか。コミカル、PV、追跡、クールアクション、フォトリアル時間停止、Vlogなど。

混入禁止:
  そのモードに混ぜてはいけない表現は何か。

参照固定:
  何を変えないようにしているか。

参照ロール:
```

## 実務チェックリスト

Seedanceに投入する前に、次を確認します。

- 主表現モードを1つ選んでいる。
- 補助表現は最大1つに絞っている。
- `DO NOT MIX` で、主表現と衝突する語彙を禁止している。
- コミカルなシーンにフォトリアル時間停止スリラー、犯罪映画、実写的破壊、クール殺陣の語彙を混ぜていない。
- 参照画像の固定指示がある。
- 複数参照の場合、画像ごとの役割スロットがある。
- 絵コンテやラフ参照がある場合、撮影指示として読むことが明記されている。
- キャラクター外見を変えない指示がある。
- 尺が明記されている。
- カメラ方針が明記されている。
- 固定カメラ、カメラ切替、追従、微振動、ロングテイクのどれを使うか決まっている。
- 映像質感が、劇場アニメ、ゲームシネマティック、Vlogなどとして明記されている。
- 秒単位のタイムラインがある。
- 必殺技や短いアクションの場合、フェーズ設計がある。
- 各時間帯に目的がある。
- 重要な感情変化に停止ポーズがある。
- 表情ラッシュを入れる見せ場が決まっている。
- ダンス/PVの場合、完全停止か微振動ホールドかが決まっている。

## 現時点の結論

BP-001の強さは、細かい言葉の量そのものではなく、情報の順番と解像度にあります。追加資料からは、そこへカメラ設計、グラフィックレイヤー、複数キャラ管理、音声同期、目的物管理、変換ギミックの因果連続性、撮影質感の指定を加えることで、よりPV的で複雑な短尺映像にも対応できることが分かりました。

有効な順番は、まず「表現モード」と「混入禁止」を決め、その後で「変えないもの」と「参照ロール」を固定し、次に「アンカー画像の解釈」「画面世界」「撮影質感」「カメラ方針」を決め、その上で「目的物」「因果連続性」「秒単位の演技またはフェーズ」を書き、最後に「品質優先順位」と「回避事項」を再提示することです。

Seedance向けの動画プロンプトでは、単に出来事を並べるのではなく、観客が読み取るべき感情と、モデルが描くべき身体部位を一致させます。特に、予備動作、停止ポーズ、副次運動、表情部位の分解を入れることで、短尺でも意図した芝居が出やすくなります。ダンスやPVでは、カメラの拍同期、常時カメラ目線、2Dエフェクトのレイヤー管理、決めポーズ時の微振動ホールドを使うと、短尺映像としての密度と見飽きなさが上がります。アクションや魔法では、溜め、解放、残響のフェーズ、攻撃や粒子が次ショットへ残る因果連続性、安全な無力化方法を明確にすると、派手さと読みやすさを両立できます。Vlog風では、不完全な手持ち、フォーカス揺れ、露出揺れ、自然音を品質として指定し、映画的に整えすぎないことが重要です。

最も重要なのは、表現をパッチワークしないことです。コミカルなキャンプ、食べ物争奪、かわいい逃げ回りでは、クールな犯罪スリラー、フォトリアル時間停止、実写的剛体破壊の語彙を混ぜず、表情、間、目的物、かわいい自滅、無傷のオチを優先します。必要な表現だけをモード単位で選び、別モードの語彙は `DO NOT MIX` で明示的に止めます。
